# Task App

ガントチャート＋イナヅマ線によるプロジェクト管理ツール。

## 主な機能

- **プロジェクト管理**: プロジェクトの作成・編集・削除
- **タスク管理**: 階層構造（親子）、マイルストーン、担当者割当、進捗管理
- **ガントチャート**: SVG自作の描画エンジン、ズーム切替（日/週/月）
- **イナヅマ線**: 進捗線の可視化、複数スナップショットの重ね表示
- **自動スナップショット**: アプリ起動時に未記録週を自動生成（過去進捗を履歴から復元）
- **メンバー管理**: 担当者の登録、ガント上での色分け
- **フィルタ・折りたたみ**: 担当者フィルタ、階層タスクの展開/折りたたみ

## セットアップ

```bash
pip install -r requirements.txt
```

## 起動

プロジェクトルートから:

```bash
uvicorn backend.main:app --reload
```

ブラウザで http://localhost:8000 を開く。

## 使い方

### 1. メンバーを登録

`/members` でプロジェクトに参加するメンバーを登録します。色を設定すると、ガントチャート上で担当タスクの色分けに使われます。

### 2. プロジェクトを作成

`/` から新規プロジェクトを作成します。開始日・終了日は任意で、未設定ならタスクの日付から自動算出されます。

### 3. タスクを追加

プロジェクト詳細画面で「+ タスク追加」からタスクを作成します。

- **階層構造**: 「親タスク」を選んで子タスクを作成。親タスクはバーが描画されず、グループ見出しとして表示
- **マイルストーン**: チェックすると◆マークで表示
- **担当者**: 割り当てるとガント上でメンバーの色になる
- **並び順**: 同階層内では開始日順で自動ソート

### 4. 進捗を更新

タスクをクリックすると編集モーダルが開きます。進捗を変更すると履歴が自動記録されます。

### 5. イナヅマ線を見る

- 起動時に毎週月曜日のスナップショットが自動生成されます
- 「今すぐスナップ」ボタンで任意のタイミングで作成可能
- 「スナップショット管理」で過去日付の指定作成や個別削除も可能
- 古いイナヅマ線は薄く、最新は濃く表示されます

#### イナヅマ線の読み方

- **左に凹む** → 遅延
- **右に飛び出す** → 先行
- **基準日線上に乗る** → 予定通り

### 6. フィルタと折りたたみ

- **担当者フィルタ**: 特定の担当者のタスクだけ表示。未割当のタスクのみの絞り込みも可能
- **折りたたみ**: 親タスクの「▼」をクリックして子タスクを非表示に
- **全展開/全折りたたみ**: ボタン一発で全階層を制御

## データモデル

- **projects**: プロジェクト（owner_id は将来の認証用）
- **members**: メンバー（担当者）
- **tasks**: タスク（階層構造、マイルストーン対応）
- **task_progress_history**: 進捗履歴（スナップショット復元の根拠データ）
- **progress_snapshots / snapshot_task_progress**: イナヅマ線データ（ヘッダ＋明細）

## 設計の特徴

### 自動スナップショット生成

アプリ起動時に各プロジェクトの未記録週（毎週月曜日基準）を検出し、`task_progress_history` から「その時点の進捗」を正確に復元してスナップショットを生成します。cron等の外部依存なしで動作します。

### SVG ガント描画

ガントチャートはライブラリを使わずSVGで自作しています。設計原則:

- **データ → レイアウト計算 → 全描画** を毎回ゼロから実行
- 座標は全て関数で算出（Y=index×行高さ、X=日付差×日幅）
- タスクの追加・編集・削除でズレない宣言的な描画
- イナヅマ線も同じレイアウト結果を参照するため整合性が保たれる

### 認証対応の設計

現状は認証なしですが、各リソースに `owner_id` を持たせる構造になっており、`Depends(get_current_user)` を追加するだけで認証可能な設計です。

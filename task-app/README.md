# Task App

ガントチャート＋イナヅマ線によるプロジェクト管理ツール。

## 主な機能

- **ユーザー認証**: ユーザー登録・ログイン（セッションCookie）。プロジェクトはユーザーごとに分離
- **プロジェクト管理**: プロジェクトの作成・編集・削除、完了フラグによるアーカイブ
- **統合プロジェクト**: 複数プロジェクトの進捗を1枚のガントにまとめる閲覧専用ビュー。1プロジェクト=1バーで、進捗点は「最も遅れているタスク」基準
- **タスク管理**: 階層構造（親子）、マイルストーン、担当者割当、進捗管理
- **親タスクの自動計算**: 親の進捗は子の平均から自動算出。全子タスク完了で親も完了になる
- **タスク移動**: タスクを子孫ごと別プロジェクトへ移動
- **ガントチャート**: SVG自作の描画エンジン、ズーム切替（日/週/月）、今日線、週末・祝日の網掛け
- **ドラッグ操作**: バーをドラッグして日付移動、端をつまんで開始日/期日を伸縮
- **遅延ハイライト**: 計画進捗（経過日数割合）を下回るタスクを赤枠・遅延バッジで自動強調
- **タスク依存関係**: 先行→後続を登録し、ガント上に矢印表示（循環は自動検出して拒否）
- **フロー図**: 依存関係を作業フローチャートとして可視化（担当者・必要日数付き）
- **タスクテンプレート**: 定型のタスク一式＋必要日数＋先行タスクを登録し、開始日（または終了日から逆算）だけで日付を自動計算（土日祝スキップ対応）
- **PNG出力**: ガントチャート・フロー図を画像として保存（報告資料用）
- **イナヅマ線**: 進捗線の可視化、複数スナップショットの重ね表示
- **自動スナップショット**: アプリ起動時に未記録週を自動生成（過去進捗を履歴から復元）
- **ToDoリスト**: 全プロジェクト横断で「着手すべきタスク」を説明・残日数（営業日併記）付きで一覧表示
- **メンバー管理**: 担当者の登録、ガント上での色分け
- **フィルタ・折りたたみ**: 担当者フィルタ、完了タスクの表示切替ボタン、階層タスクの展開/折りたたみ（状態はブラウザに保存）

## セットアップ

```bash
cd task-app
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## 起動

起動スクリプトを使う（`.venv` / `venv` があれば自動で有効化されます）:

```bash
# Linux / macOS
./start.sh

# Windows
start.bat
```

ブラウザで <http://localhost:18234> を開く。API仕様書（Swagger UI）は <http://localhost:18234/docs> 。

手動で起動する場合（`task-app/` から）:

```bash
python -m uvicorn backend.main:app --host 127.0.0.1 --port 18234
```

開発時は `--reload` を付けるとコード変更が自動反映されます。

SQLiteデータベースは初回起動時に `data/tasks.db` として自動生成されます。

## 使い方

### 0. ユーザー登録・ログイン

初回アクセス時は `/login` にリダイレクトされます。「新規登録」からユーザーを作成してください
（認証導入前から使っていたデータは、**最初に登録したユーザー**の所有になります）。
ヘッダーにユーザー名とログアウトリンクが表示されます。

### 1. メンバーを登録

`/members` でプロジェクトに参加するメンバーを登録します。色を設定すると、ガントチャート上で担当タスクの色分けに使われます。

### 2. プロジェクトを作成

`/` から新規プロジェクトを作成します。開始日・終了日は任意で、未設定ならタスクの日付から自動算出されます。

### 3. タスクを追加

プロジェクト詳細画面で「+ タスク追加」からタスクを作成します。

- **階層構造**: 「親タスク」を選んで子タスクを作成。親タスクはバーが描画されず、グループ見出しとして表示
- **マイルストーン**: チェックすると◆マークで表示
- **担当者**: 割り当てるとガント上でメンバーの色になる
- **並び順**: 同階層内では開始日順で自動ソート
- **別プロジェクトへ移動**: 編集モーダルの「別プロジェクトへ移動」で、子タスクごと移動（移動後は最上位タスクになる）

### 4. 進捗を更新

タスクをクリックすると編集モーダルが開きます。進捗を変更すると履歴が自動記録されます。

親タスク（子を持つタスク）の進捗は子タスクの平均から自動計算されます（編集モーダルでは入力不可）。
**全ての子タスクが100%になると親も自動的に完了**し、子が未完了に戻れば親の完了も解除されます。

日付の変更はガント上のドラッグでも可能です:

- **バー中央をドラッグ** → タスク全体を移動（期間はそのまま）
- **バーの左右端をつまんでドラッグ** → 開始日/期日を伸縮
- **マイルストーン（◆）をドラッグ** → 日付を移動

計画より遅れているタスク（実績進捗 < 経過日数の割合）は、バーが赤枠になり進捗欄に⚠が付きます。

### 4b. 依存関係とフロー図

タスク編集モーダルの「先行タスク」で、そのタスクの前に終わっているべきタスクを登録できます。

- ガント上に先行→後続の矢印が描画されます
- 循環する依存（A→B→A）はサーバ側で検出され登録できません
- ツールバーの「フロー図」で、依存関係を作業フローチャートとして表示できます（担当者・必要日数・進捗付き。カードクリックでタスク編集へ）

### 4c. テンプレートからタスク一式を作成

「テンプレート」ページで定型のタスク一式（タスク＋子タスク＋説明＋必要日数＋先行タスク）を
登録しておくと、プロジェクト画面の「テンプレートから追加」で **日付を1つ指定するだけ** で
全タスクの日付が自動計算されます。

- **開始日から計算**: 指定日から前方に日程を配置
- **終了日から逆算**: 「この日までに完了」から逆算して各タスクの開始日を決定
- **先行タスク**: テンプレート内で「先行」を設定すると、先行タスクの完了後に開始するよう配置。
  先行が重ならないタスクは**並行**に配置される。適用時にタスクの依存関係としてコピーされ、フロー図にそのまま反映
- 先行の指定がなければ上から順に直列配置。子タスクを持つ親は子の期間全体をカバー
- 「土日祝を除いて日程を計算する」がデフォルトON（必要日数=営業日として扱う）
- 祝日は日本の祝日（振替休日・国民の休日含む）を自動計算

### 4d. PNG出力

- ガントチャート画面の「PNG出力」: タスク名・担当・進捗の一覧付きでチャートを画像保存
- フロー図画面の「PNG出力」: フローチャートを画像保存

### 5. イナヅマ線を見る

- 起動時に毎週月曜日のスナップショットが自動生成されます
- 「今すぐスナップ」ボタンで任意のタイミングで作成可能（同日分は上書き）
- 「スナップショット管理」で過去日付の指定作成や個別削除も可能
- 古いイナヅマ線は薄く、最新は濃く表示されます（表示本数は変更可能）

#### イナヅマ線の読み方

- **左に凹む** → 遅延
- **右に飛び出す** → 先行
- **基準日線上に乗る** → 予定通り

### 6. フィルタと折りたたみ

- **担当者フィルタ**: 特定の担当者のタスクだけ表示。未割当のタスクのみの絞り込みも可能
- **完了タスクの表示切替**: 進捗100%のタスクはデフォルトで非表示。「完了タスク: 非表示/表示中」ボタンで切り替え（設定はブラウザに保存）
- **折りたたみ**: 親タスクの「▼」をクリックして子タスクを非表示に（状態は localStorage に保存され、再訪時も維持）
- **全展開/全折りたたみ**: ボタン一発で全階層を制御

### 7. ToDoリストで今日やることを確認

`/todo` で全プロジェクトを横断して「進捗100%未満かつ開始日が今日以前」のタスクを一覧表示します。

- タスクの説明、期限までの残日数（暦日＋営業日）付き、期限が近い順に表示
- 計画進捗より遅れているタスクには赤い「遅延」バッジ
- 担当者で絞り込み可能
- 子を持つ親タスク（グループ見出し）と完了済みプロジェクトのタスクは除外
- タスクをクリックするとプロジェクト画面に飛び、編集モーダルが開く

### 8. プロジェクトを完了にする

プロジェクト一覧の「完了にする」で完了フラグを立てられます。完了済みプロジェクトは一覧・ToDoから隠れます（「完了済みも表示」で確認可能）。

### 9. 統合プロジェクトで全体を俯瞰する

`/portfolios`（ナビの「統合」）で複数プロジェクトをまとめた**閲覧専用**のガントビューを作れます。

- 名前と対象プロジェクトを選んで保存。複数の統合プロジェクトを作成可能
- **1プロジェクト = 1本のバー**（配下タスクの最早開始日〜最遅終了日）
- **進捗点は最も遅れているタスク基準**: 各未完了タスクの進捗到達日（開始日＋期間×進捗%）を
  計算し、最も遅れている到達日をそのプロジェクトの進捗点にする（イナヅマ線の凹みと同じ考え方）。
  開始前の未着手タスクは「順調」扱い、全タスク完了なら100%
- 基準日は**今日**。左ペインに「n日遅れ / 順調 / n日先行」と最遅タスク名（ツールチップ）を表示
- この画面から編集はできない。バーやプロジェクト名のクリックで各プロジェクトのガント画面へ移動
- PNG出力対応

## データモデル

- **users / user_sessions**: ユーザー（PBKDF2ハッシュ）とログインセッション（Cookieトークン）
- **projects**: プロジェクト（`is_completed` 完了フラグ、`owner_id` でユーザーごとに分離）
- **portfolio_projects / portfolio_project_items**: 統合プロジェクト（名前＋対象プロジェクトの並び。ユーザーごとに分離）
- **members**: メンバー（担当者）。全ユーザー共有で、タスクの `assignee_id` から参照される
- **tasks**: タスク（`parent_task_id` による階層構造、マイルストーン対応。親の進捗は子から自動計算）
- **task_dependencies**: タスク依存関係（`predecessor_id` → `successor_id`、同一プロジェクト内のみ）
- **task_templates / task_template_items / template_item_dependencies**: タスクテンプレート（ヘッダ＋階層アイテム＋必要日数＋アイテム間依存）
- **task_progress_history**: 進捗履歴（追記専用。スナップショット復元の根拠データ）
- **progress_snapshots / snapshot_task_progress**: イナヅマ線データ（ヘッダ＋明細）

## 設計の特徴

### 自動スナップショット生成

アプリ起動時に各プロジェクトの未記録週（毎週月曜日基準）を検出し、`task_progress_history` から「その時点の進捗」を正確に復元してスナップショットを生成します。cron等の外部依存なしで動作します。

### SVG ガント描画

ガントチャートはライブラリを使わずSVGで自作しています。設計原則:

- **データ → レイアウト計算 → 全描画** を毎回ゼロから実行
- 座標は全て関数で算出（Y=index×行高さ、X=日付差×日幅）
- タスクの追加・編集・削除でズレない宣言的な描画
- イナヅマ線も同じレイアウト結果を参照するため整合性が保たれる

### 軽量マイグレーション

`database.py` の `_apply_migrations()` が起動時にカラム不足を検出して `ALTER TABLE` を実行します。Alembic等は不使用。カラム追加時はここに追記します。

### 認証とデータ分離

- パスワードは PBKDF2-HMAC-SHA256（ソルト付き）でハッシュ化。外部ライブラリ不要
- ログインセッションは `user_sessions` テーブル＋HttpOnly Cookie（有効期限30日）
- プロジェクト（とその配下のタスク・スナップショット・依存関係）は `owner_id` でユーザーごとに分離
- メンバーとテンプレートは全ユーザー共有
- 認証導入前のデータは最初に登録したユーザー（id=1）の所有になる

### 祝日・営業日計算

日本の祝日（ハッピーマンデー・春分秋分・振替休日・国民の休日）をアルゴリズムで算出します
（1980〜2099年で有効）。**同一アルゴリズムが2箇所にあります**:
フロント `frontend/js/util.js` の `jpHolidays()` とバックエンド `backend/services/jp_holidays.py`。
変更時は両方を更新してください。

### テンプレートの日程自動計算

- 先行タスク指定なし → 末端タスクを並び順に直列配置
- 先行タスク指定あり → 依存グラフをトポロジカル順に処理し、先行の完了後に開始（並行あり）
- 終了日指定時は逆順に処理して「この日までに完了」から逆算
- 親アイテムは子孫の期間全体をカバー。営業日/暦日は選択可能

### 統合プロジェクトの進捗集計

`GET /api/portfolios/{id}/summary` がプロジェクトごとに以下を計算します（基準日=今日）:

- バー期間 = 末端タスクの最早開始日〜最遅終了日
- 進捗到達日 = タスク開始日＋期間×進捗%。未完了タスクのうち最遅の到達日をプロジェクトの進捗点とする
- 開始前かつ未着手のタスクは基準日上（順調）とみなす（イナヅマ線と同じルール）
- `delay_days` = 基準日 − 進捗点（正=遅延、負=先行）、`bottleneck_task` = 最遅タスク名

フロントは各プロジェクトを疑似タスク、進捗点を「基準日=今日の疑似スナップショット」に変換して
`gantt.js` をそのまま再利用しています。
